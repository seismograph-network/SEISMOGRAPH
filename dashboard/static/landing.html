The Problem

Status pages track uptime. No one tracks semantics.

When an LLM provider silently ships a new checkpoint—different RLHF, updated safety filters, quantization changes—your monitoring stays green. Latency is fine. Uptime is 100%. But your application breaks in ways no alert will catch.

What existing tools see
Uptime · Latency · Token counts
Provider status pages and APM dashboards report infrastructure health. They are blind to a model that responds fast but has silently shifted its reasoning, instruction-following, or JSON output fidelity.
What SEISMOGRAPH sees
Semantics · JSON fidelity · Output drift
Deterministic temperature-0 canary probes measure what the model means, not just whether it responded. A cross-org quorum of ≥2 independent observers must agree before any public drift alert fires—single-org noise is never promoted.

How It Works

Five minutes from install to your first signal.

01
Install the probe
pip install seismograph-probe — a 33 KB wheel with no server dependencies. Supports OpenTelemetry gen_ai.* conventions out of the box.
02
Run temperature-0 canary probes
Deterministic prompts are sent to your LLM API. SHA-256 hashes and aggregate statistics—never raw text—are recorded per probe run.
03
Signals are DP-noised and Ed25519-signed
Differential privacy (ε=2.0 Laplace) is applied before any data leaves the probe. Each batch is signed for Sybil resistance. Your prompts and outputs never leave your machine.
04
CUSUM detector + cross-org correlation
A CUSUM change-point detector tracks each model’s signal stream. A quorum of ≥2 independent organizations must agree before any public drift alert is raised. One org’s noise stays private.
05
Webhook fires before your users notice
Private fleet alerts trigger your webhook immediately—integrate with your CI/CD to gate rollouts. The public Model Weather dashboard shows community-wide drift status for the top model tuples.
