## 5. Architectural decisions

### REQ-PRIV-002 — Ed25519 probe identity (APPROVED 2026-06-10)

**Decision:** Each probe installation generates a unique Ed25519 keypair at
install time. The probe signs every outbound `SignalBatch` with its private
key. The central ingestion engine holds only the public key and builds a
reputation score against that key over time. The engine never learns the
underlying organisational identity from the key itself.

**Rationale:** This achieves pseudonymous federation -- the engine can detect
Sybil attacks (new keys with no reputation) and down-weight low-reputation
contributors without requiring org identity disclosure. It preserves the
Privacy by Construction invariant while enabling trust calibration.

**Approved by:** Tatiana (P0-004 session, 2026-06-10)

**Implementation status:** Phase 0 stub in place (`gateway/auth.py`
`verify_signature()` returns `True`; `gateway/auth.py` carries
`# TODO(Phase 2): Implement Ed25519 verification (REQ-PRIV-002)`).
Cryptographic wiring deferred to Phase 2 to maintain Phase 0 velocity.

**Reference:** `gateway/auth.py`, `gateway/ingest.py`, `gateway/ingestion.py`

## 7. Methodology note — suggested improvement

**Finding:** The autopep8 + sed pipeline corrupted multi-byte UTF-8
characters in Python source files, truncating file content at byte 79
rather than character 79.

**Suggested improvement:** Establish a pre-commit hook using `ruff`
(which is UTF-8 aware and handles line-length on character boundaries)
rather than pycodestyle + autopep8 + sed. Ruff is also significantly
faster and consolidates lint + format in one tool. Add a `pyproject
.toml` at repo root with `[tool.ruff]` config set to `line-length = 79`
and `target-version = "py311"`. This should be the first `pyproject
.toml` task in P0-002.

### Known limitations (Phase 1)

- Ed25519 signature verification is a stub. `verify_signature()` returns True
  unconditionally. Phase 2 must implement REQ-PRIV-002 before any public
  deployment (SECURITY: Sybil attacks are possible until this is wired).

- CUSUM state is ephemeral (in-memory). Gateway restart resets all CUSUM
  accumulators. The `bootstrap_detector()` function restores baseline from
  stored telemetry signals, but drift accumulation since last flush is lost.

- AgreementScorer state is also ephemeral. Pending quorum contributors are
  cleared on gateway restart. Cross-session quorum accumulation is a Phase 2
  distributed state problem (Redis/ClickHouse).

- SQLite is not production-grade for multi-writer scenarios. ClickHouse
  migration is the Phase 2 storage milestone.

- DP sequential composition is not tracked across flush windows. Total epsilon
  exposure per probe per day is unbounded in Phase 1. REQ-PRIV-010 (Phase 2).

- Canary suite (CANARY_SUITE_V1) contains 3 prompts. Phase 2 should expand
  to the full 200-prompt corpus with provider ToS compliance verification per
  target API.

- No authentication on GET /v1/weather or GET /. Public by design for Phase 1
  "model weather" use case. Phase 2 multi-tenant isolation requires RBAC.
